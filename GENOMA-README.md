## Theory

The GENOMA system treats the ARKADU archive as a **living organism**:

- **Files = Genes (Codons)** - Each file is a functional unit of information
- **Species = Genomes** - File types (PNG, MP4, etc.) are genetic sequences
- **Archive = Organism** - The complete collection is a living digital entity
- **Transclusion = Expression** - Connections across views are genetic expressions

## Codon Types

### Media Codons (Exons - Expressed)
- `PNG` - Image files (schooling prey)
- `JPG` - JPEG images
- `MP4` - Video files (apex predator)
- `WAV` - Audio files
- `JSON` - Metadata files
- `PY` - Python scripts (tool chains)

### Structural Codons
- `DIR` - Directory containers
- `DEPTH` - Hierarchy level markers
- `SPECIES` - File type groupings

### Functional Codons
- `TETHER` - Cross-view connections
- `FLAG` - Marked nodes
- `TAG` - Annotated nodes
- `MEASURE` - Comparison operations

### Genetic Codons (Introns - Stored but not expressed)
- `CMT` - Commentary/notes
- `LINK` - Ekphrasis chain links
- `TOOL` - Tool chain references

## Usage

### 1. Sequence All Species Genomes

```bash

```

This reads `sys/primitive.jsonl` and generates `sys/genoma-sequences.json` containing:
- Complete genome for each species (file type)
- All ekphrasis chain sequences
- Metadata and statistics

### Transclusion
Expressing same gene in multiple views (Voronoi + Sediment)

### Recombination
Combining two genomes at crossover point

### Mutation
- **Point**: Change a codon type
- **Insertion**: Duplicate a codon
- **Deletion**: Remove a codon

## Integration with Double Barrel

The GENOMA system integrates with the double-barrel interface:

1. **Flagging a node** creates a TETHER codon
2. **Tagging a node** creates a TAG codon  
3. **Measuring** creates a MEASURE codon
4. **Exporting** generates complete genome sequences

## File Origins

Each codon tracks its origin:
- `operative_ekphrasis` - From ekphrasis chains
- `tool_chain_output` - Generated by tools
- `metadata` - Structural/descriptive
- `source_material` - Original artifacts
- `excavated_artifact` - Discovered files
